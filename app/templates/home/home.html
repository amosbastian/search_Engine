SERP

The SERP of our search engine displays:
- a bar graph acting as a timeline showing the amount of articles returned per year. The bargraph is made using Google Charts.
- a wordcloud containing the most common words found in the titles of the returned articles. This wordcloud is not very reliable because of this. Trying to search in the bodies instead of the titles gave connection timeouts. The wordcloud is made using wordcloud2.js from https://github.com/timdream/wordcloud2.js/
- A list of result articles showing the the title, some of the body, the date of the article and the type of article.

Shortcomings

- Unfortunately, due to time constraints we weren't able to conveniently save the user input when displaying search results: all user inputs in the fields will be wiped and the input fields will be returned to their default values.
- As mentioned before, we were also unable to get the wordcloud properly displaying the most common terms of the search results.
- The displayed search results are limited as we did not want to flood the page with articles. We did not take enough time to implement a dynamic way to let the user browse through multiple pages of results

Simple Search

Simple Search contains a single input bar in which the user can type in a query. This query is used to search in both the titles and bodies of the articles in the index. Using the query type dis_max, the query returns the union of the results from the titles and from the bodies.

Advanced Search

Advanced Search has four input bars. The first is the input bar for the query string used to search in the titles of the articles. The second is for the query string used to search in the bodies. The third and fourth are for specifying a lower and upper bound for the time period in which the user wants to search.

Unlike in simple search, advanced search uses boolean search with the search type "should". With this, the user specifies that the returned results SHOULD contain certain terms, but they don't have to. Both the query strings don't HAVE to match, but this search method values articles higher if both the terms for the title and the terms for the bodies match.

Also unlike simple search, in advanced search the user is allowed to leave fields open. Leaving both input bars empty will return all articles in the specified time period. If the lower bound is left empty, the search will only return articles up until the upper bound, and vice versa for leaving the upper bound empty. Leaving everything empty will return all articles

Faceted Search

Faceted Search has one input bar and four checkboxes, one for each of the four types of articles. Checking the boxes for the facets allows the search to return articles belonging to that facet. Articles belonging to unchecked facets are filtered out. Faceted Search uses the same searching method as Simple Search, except for the filtering part
